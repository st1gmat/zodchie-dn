Design direction chosen for the "–ó–æ–¥—á–∏–µ" plumbing-store landing page:
- **Visual style (updated 2026-06-22):** light palette, e-commerce-catalog-style structure inspired by https://www.santehcentr.com/ ‚Äî white/light-gray background, dark text, warm brass/copper accent (`--accent` etc. in `src/app/globals.css`, now inverted from the original dark theme). Header has: top utility bar (address/hours/phone), main row (logo, nav, favorites+cart icon links, "–ü–æ–∑–≤–æ–Ω–∏—Ç—å" button) ‚Äî no category row in the header. Superseded the earlier "dark premium, deliberately not like competitors" direction from 2026-06-20 ‚Äî user changed their mind after seeing santehcentr.com and explicitly asked to adopt its structure + light colors.
- **Category nav placement (corrected 2026-06-22):** on santehcentr.com the category list is a narrow left **sidebar** (`<aside class="sidebar col-lg-2">`, confirmed by reading the raw HTML ‚Äî not a full-width block as an earlier WebFetch summary wrongly suggested) sitting next to the hero/carousel (`col-lg-10`), not inside or under the header. We matched this: `src/components/CatalogSidebar.tsx` (vertical category list, `lg:`-only, hidden on smaller screens) renders inside `Hero.tsx` as a flex sibling to the hero text column. Don't trust a single WebFetch text summary for pixel/layout-level structure questions again ‚Äî grep the raw HTML (Bootstrap col-* classes, aside/sidebar markup) when the user pushes back on a layout claim.
- **Cart/favorites icons:** currently decorative-only (link to `#catalog`, `title` tooltip "–ø–æ—è–≤–∏—Ç—Å—è –ø–æ–∑–∂–µ") ‚Äî no real cart/wishlist logic exists yet; don't treat the icons as a sign that cart functionality is implemented.
- **Content:** placeholder copy/icons for now ‚Äî no real product photos or logo yet. Don't fabricate fake stats/numbers (e.g. "10 years on market", review counts); generic marketing claims (delivery, consultation) are fine, factual-sounding numbers are not.
- **Build approach:** stepwise/iterative, not one giant page at once. First slice built (2026-06-20): `Header`, `Hero`, `CategoryGrid` (8 placeholder categories) in `src/components/`, config in `src/lib/site.ts`. Restyled to light/santehcentr-structure on 2026-06-22 (same components, no new sections).
- **DB layer wired (2026-06-22):** moved categories off the static `site.ts` array onto Postgres via Prisma 7. Schema: `Category` (self-relation parent/children + `icon` string keyed to `CategoryIcon.tsx` map like "faucets"/"toilets"), `Product`, `ProductImage` in `prisma/schema.prisma`. Front-end now reads from DB: `src/lib/categories.ts` `getCategories()` (top-level, ordered) ‚Üí `page.tsx` is an async server component that passes `categories` into `Hero`‚Üí`CatalogSidebar` and `CategoryGrid` as props. The static `categories`/`Category` export was removed from `site.ts` (siteConfig contacts/legal stay static there). Gotchas for this stack: Postgres runs in Docker on **host port 5433** (5432 was taken by another project's `ms_pg_sql` container) ‚Äî see `docker-compose.yml`/`.env`; Prisma 7's `prisma-client` generator outputs TS to `src/generated/prisma`, import `PrismaClient` from `@/generated/prisma/client` (no index); it **requires a driver adapter** ‚Äî `new PrismaClient({ adapter: new PrismaPg({ connectionString }) })` (`@prisma/adapter-pg` installed), bare `new PrismaClient()` throws; seed runs via `npx tsx` (config `seed`) and needs `import "dotenv/config"`; re-run `prisma generate` after schema field changes or the client validation lags. `/` currently prerenders **static** at build (categories baked in) ‚Äî add `revalidate`/`force-dynamic` once the admin panel can edit categories at runtime.
- **Public catalog pages built (2026-06-22):** `/catalog` (category index), `/catalog/[slug]` (category + product grid), `/catalog/[slug]/[product]` (product detail). Data accessors in `src/lib/catalog.ts` (`getCategoryDetail`, `getProductDetail` ‚Äî the latter validates the product actually belongs to `categorySlug`, else `notFound()`, to avoid duplicate URLs); price formatting in `src/lib/format.ts` (whole rubles, `ru-RU` nbsp). Components: `ProductCard.tsx`, `Breadcrumbs.tsx`. `Header` was moved out of `page.tsx` into the root `layout.tsx` so it renders on every route (and `<main>` lives there now); home `page.tsx` returns just `<Hero>`+`<CategoryGrid>`. All internal nav switched from `#catalog` hash anchors to `next/link` real routes (Header logo/nav/icons, Hero CTA, `CategoryGrid` cards, `CatalogSidebar`). Seeded **32 demo products** (4 per category, a couple `inStock:false`) ‚Äî clearly placeholder nomenclature until real/1C data; products have no photos so cards fall back to `/images/product-placeholder.png` (WooCommerce-style no-image placeholder the user supplied, in `public/images/`). The `[slug]` and `[slug]/[product]` routes are **dynamic (SSR on demand)** ‚Äî no `generateStaticParams` ‚Äî so they reflect DB edits immediately (unlike static `/` and `/catalog`). Next 16 dynamic-route notes: `params` is a Promise (`await params`), type pages with the `PageProps<'/route'>` global helper, `notFound()` from `next/navigation`.
- **Admin panel built (2026-06-22):** single-admin auth ‚Äî no user table, password compared (`bcryptjs`) against `ADMIN_PASSWORD_HASH`, session is a `jose` HS256 JWT in an httpOnly cookie `admin_session` signed with `SESSION_SECRET`. Helpers in `src/lib/auth.ts` (`verifyPassword`, `createSession`, `destroySession`, `isAuthenticated`, `requireAdmin`); `prisma/hash-password.ts` prints a hash (`npx tsx prisma/hash-password.ts <pwd>`). **Local dev admin password is `zodchie-admin`** (hash sits in gitignored `.env`). Pages: `/admin/login` (public) + a `(panel)` route group whose `layout.tsx` calls `requireAdmin()` and renders admin chrome ‚Äî dashboard `/admin`, `/admin/categories` (+`[id]` edit, inline create), `/admin/products` (+`new`, `[id]` edit). Mutations are Server Actions in `src/app/admin/actions.ts` (`"use server"`, every action re-checks `requireAdmin()`, validates with `zod`, then `revalidatePath("/", "layout")` so the static storefront refreshes). No middleware/proxy used (Next 16 renamed middleware‚Üí**proxy**; guard via layout + per-action check instead). The storefront `Header` hides itself on `/admin` via `usePathname()` (avoided a route-group refactor of the public pages, which was blocked by a Windows file lock from the running dev server). Slug is set on create and **left unchanged on edit** (keeps URLs stable). NOT done in admin yet: managing product images & `ProductAttribute` specs (still seed-only), login rate-limiting, multiple admins. The fuller storefront (cart, comparison) like santehcentr.com is still deferred.
- **Still to build:** advantages/"why us" section, "about company" (using real legal info from [[project-business-info]]), contacts section with map, footer with legal/INN details. Plus the admin panel above. Product detail page now has a `ProductGallery` (client, thumbnails only when >1 image) and a "–•–∞—Ä–∞–∫—Ç–µ—Ä–∏—Å—Ç–∏–∫–∏" spec list from the new `ProductAttribute` model (relational, mirrors `ProductImage`; seeded ~4 demo specs per category via `specsByIcon`, reset+recreated each seed for idempotency). Still no cart. 1C nomenclature import still undecided per README/CLAUDE.md.
